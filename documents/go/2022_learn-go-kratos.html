# 学习go-kratos

# Kratos简介

Kratosopen in new window 一套轻量级 Go 微服务框架，包含大量微服务相关框架及工具。

# 创建项目

根据官方文档的方式来创建项目open in new window。

# api目录

apiopen in new window目录里面使用Protobuf定义接口信息。
在项目跟目录执行命令make api生成接口信息。
里面还有错误定义文件erroropen in new window。

# cmd目录

cmd目录里面是程序的入口文件main。 kratos使用谷歌开源wire管理依赖（依赖注入open in new window） 修改wire.go里面wireApp函数的参数，再修改main使用的wireApp函数，可提供依赖给wire。
然后在internal目录里面，有的包会使用var ProviderSet = wire.NewSet(func_name...)来设置需要依赖注入的函数。
最后到cmd/xxxx目录下执行wire，就会自动生成wire_gen.go文件。main使用的wireApp函数就是这里面的。

# internal目录

internal目录里面编写项目私有的代码。

# server目录

server目录里面是创建服务的代码

# service目录

service目录里面写的接口的实现，就是MVC的C控制层。

# biz目录

biz目录里面写的是业务逻辑，就是逻辑层啦。
Repository的接口定义也是写这边。

# data目录

data目录里面编写是获取数据的一些实现。
也就是Repository的实现

# conf目录

conf目录就是写配置信息。
也是使用Protobuf定义配置信息。
在根目录上执行make config，会生成配置信息go代码。
configs目录下有config.yaml，可写配置信息。

# 最后

我的开发流程是这样子：

  1. 看需求设计api接口
  2. 编写api接口定义
  3. 编写实现api接口的函数，并调用业务逻辑接口
  4. 编写业务逻辑接口，并写Repository接口定义
  5. 编写实现Repository
